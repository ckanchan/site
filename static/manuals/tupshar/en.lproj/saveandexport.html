Save and export a document

To save a document, use the File -> Save (âŒ˜+S) command. In addition, Tupshar offers two export options:

  • Export as Text: This option writes out the cuneiform, transliteration, normalisation, word-by-word translation and document translations into a JSON object saved to a text file, which can be opened in any editor for further modification.

  • Export as Microsoft Word Document: This produces a formatted Microsoft Word .docx file, with cuneiform, transliteration, normalisation and document translation laid out one after the other with formatting applied as seen in Tupshar's text views.

Other export formats such as RTF and PDF are planned for the future.

Cuneiform fonts

Tupshar comes provided with a cuneiform font builtin to allow it to display cuneiform text. However, if your system doesn't have one installed, other programs such as text editors or Microsoft Word will not be able to display these glyphs. A number of free fonts are available for download on the web:

  • CuneiformNA, CuneiformOB and CuneiformComposite: these fonts are provided free of charge at the Oracc project and cover the Neo-Assyrian and Old Babylonian variants of the script. Tupshar uses the CuneiformNA font.

  • Noto Sans Cuneiform: part of Google's Noto font project, downloadable here. Uses Old Babylonian sign forms.

Tupshar's File Format

Tupshar uses the ocdl file format and extension to save its files. This is a minor variant of the Oracc 'cuneiform chunks and lemmas' format, described in part here. The ocdl format wraps Oracc's "cdl" and "catalogue.member" types in a single JSON object, with an additional "translation" key to store the document translation:

json {"text": { "cdl" : [CDL], "project" : "project title", "textid" : "UUID", "type" : "modern" }, "metadata" : { "ch_no" : "0", "title" : "Document Title", "id_text" : "UUID", "display_name" : "Short Title", "ancient_author" : "Document Author", "project" : "project" } }

Tupshar automatically generates a UUID for the text ID fields, and supplies defaults for others not exposed to the user in order to conform to Oracc specifications. Any of these fields can be manually edited in a text editor - as long as the result conforms to this specification it can be parsed by Tupshar.
